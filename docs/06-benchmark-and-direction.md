# 06 · 벤치마크 & 기획 방향 (living document)

> **목적**: 개선을 거듭해도 방향을 잊지 않도록 키 컨셉·벤치마크·의사결정을 기록한다.
> 밸런스·기능 결정을 바꿀 때는 반드시 해당 절과 **의사결정 로그**를 갱신한다.

## 1. 변하지 않는 키 컨셉 (건드리려면 먼저 질문)

1. **드래그 조준 + 물리 머지** — 조작은 한 손가락. 3초 안에 이해된다.
2. **무한 점수 + 0초 재시작** — 죽음은 처벌이 아니라 "여기까지 키웠다" 성과 + 재도전 초대.
3. **체인 = 보상** — 단일 머지는 기본 점수, 연쇄만이 배율(×2/×3/×4)을 올린다.
4. **우주 네온 + 귀여운 천체** — 다크 배경에서 합체 순간에 juice를 몰아준다.
5. **광고는 선택** — 부활(런당 1회)/2배 모두 유저가 "더 하고 싶어서" 보는 구조.
6. **#1(PULSE)과 겹치지 않기** — 타이밍/리듬 요소를 코어에 넣지 않는다.
7. **노바 버스트 = 시그니처** — 머지로 충전, 탭으로 발동하는 중력 펄스. 천체를 지우지 않고,
   RNG/과금이 아닌 플레이로 번다. (docs/02 §4.6, 2026-07-06)

## 3. 난이도 철학

- 긴장 곡선은 **보드가 만든다**: 초반은 여유, 쌓일수록 압박. 별도 속도 램프 불필요
  (#1과 다른 점 — 물리 머지 장르의 자연 난이도).
- 밸런싱 레버 (전부 `config.ts`): 컨테이너 폭/높이 비율, 티어 반지름 곡선, 드롭 풀 확률,
  오버플로우 유예 시간(2.0s), 체인 윈도우(1.5s).
- 목표: 첫 판에서 **달~화성**은 쉽게, **지구**는 노력, **토성+**는 계획 없이는 어렵게.
  최고 천체 도달 분포를 로컬 지표로 관찰 후 조정.
- **플레이 테스트/지표 없이 밸런스 변경 금지** (#1 원칙 계승).

## 4. 측정 (밸런스 결정의 근거) — M3에서 구현

로컬 집계 (`nova.stats` KV, 서버 불필요):
- 평균 점수 / 런 길이(초) / 런당 머지 수 / 최대 체인 / 최고 티어 도달 분포 /
  세션당 재시도 횟수 / 부활 광고 노출·수락률 / 2배 광고 수락률.
- 집계 규칙(#1 검증 패턴): 런 기록은 게임오버 시 스테이징, **부활하면 폐기**,
  다음 런 시작 시 확정. 광고 노출/수락은 즉시 기록.
