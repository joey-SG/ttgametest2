## 프로젝트 한 줄 정의

TikTok에서 바이럴 가능한 **피직스 머지-드롭 퍼즐 아케이드 "NOVA"**.
드래그 조준 + 물리 머지(별먼지→블랙홀) + 만족스러운 이펙트 + iPhone 햅틱 + 보상형 광고 부활로 "한 번 더".
빌드 결과물은 **HTML5**이며, 웹(Netlify) 데모와 TikTok Mini Game 제출 양쪽으로 배포한다.

---

## 1. Think Before Coding (먼저 생각하라)

- 가정은 **명시적으로 말하고**, 불확실하면 코딩 전에 질문한다. 조용히 추정하고 진행하지 않는다.
- 여러 해석이 가능하면 트레이드오프를 나열한 뒤 하나를 고른 이유를 밝힌다.
- **성공 기준**: 코드를 쓰기 전에 "무엇이 완료의 정의인가"를 한 문장으로 말할 수 있다.

## 2. Simplicity First (단순함이 먼저)

- 문제를 푸는 **최소 코드**. 추측성 기능·추상화·일어나지 않을 예외 처리는 넣지 않는다.
- **외부 런타임 의존성 0** — 물리도 자체 구현(원형 전용이면 충분). 물리 엔진 라이브러리 금지.
- **성공 기준**: 새 의존성/추상화 계층을 추가할 때마다 "이게 없으면 지금 무엇이 안 되는가"에
  답할 수 있다. 답이 없으면 넣지 않는다.

## 3. Surgical Changes (외과적 변경)

- 건드려야 하는 것만 건드린다. 요청받지 않은 리팩터링·인접 코드 "개선"을 하지 않는다.
- 기존 스타일 컨벤션(네이밍, 포맷)을 그대로 따른다.
- **성공 기준**: `git diff`가 요청한 변경과 직접 관련된 라인만 담는다.

## 4. Goal-Driven Execution (목표 주도 실행)

- 단계별 지시가 아니라 **검증 가능한 성공 기준**을 정의하고, 충족될 때까지 루프한다.
- **이 프로젝트의 검증 게이트 (변경 후 반드시 통과):**
  - `npm run build` 성공 (`tsc --noEmit` 포함), 빌드 산출물 총합 **< 50MB**.
  - 게임이 로컬(`npm run dev`)에서 로딩되고 첫 입력에 반응한다.
  - 코어 루프 동작: 시작 → 드래그 조준·드롭 → 머지·체인 → 오버플로우 게임오버 →
    (광고 부활) 이어하기 → 재시작 → 최고점수 갱신.
  - **물리 안정성**: 쌓인 천체가 눈에 띄게 떨리지 않고, 벽/바닥 관통이 없다.
  - `console.error`가 정상 플레이 경로에서 발생하지 않는다.
  - 60fps 목표: 바디 100개+ 상태에서 저사양 모바일 기준 프레임 드랍이 눈에 띄지 않는다.

## 게임 제품 요구사항 (완료의 정의)

- [ ] **무한 점수형**: 엔딩 없음. 점수·최고 천체를 무한히 갱신하는 구조. (블랙홀 머지=빅뱅 보너스)
- [ ] **원핑거/3초 이해**: 드래그 좌우 조준 + 릴리즈 드롭. 튜토리얼 없이 3초 안에 이해.
- [ ] **이펙트(juice)**: 머지/체인/고티어/빅뱅에 파티클·글로우·펀치·셰이크.
- [ ] **iPhone 햅틱**: 드롭/머지/체인/경고/게임오버 매핑. (구현 상세: `docs/03` §3.2 — 함정 주의)
- [ ] **광고 보상 재플레이**: 부활(런당 1회, 보드 정리) + 점수 2배. 웹은 카운트다운 폴백.
- [ ] **바이럴성**: 결과 화면에 최고 도달 천체 + 점수 + 공유.
- [ ] **#1과 차별**: 타이밍/리듬 요소를 코어 판정에 넣지 않는다.

## 플랫폼/스택 고정값 (임의 변경 금지 — 변경 시 질문)

- 결과물 포맷: **HTML5** (Canvas 2D). 무거운 엔진·물리 라이브러리 금지.
- 플랫폼 기능(광고·햅틱·저장)은 **어댑터 계층 뒤로 추상화**.
  웹 데모에서는 mock/폴백, TikTok 인앱에서는 `TTMinis.game` SDK로 스왑.
- 패키지 크기: **< 50MB** (실제 목표 < 500KB). 에셋은 100% 절차 생성.
- 밸런스 수치(티어·물리·체인·유예 시간)는 전부 `src/config.ts`.
